## Featured project

### [Datadog Observability Platform](https://github.com/smammar19/datadog-observability-platform)

> Production-grade reliability engineering platform demonstrating SLO design, burn-rate alerting, chaos engineering, and three-pillar observability using Flask and Datadog

**What it covers:**
- Custom Flask reliability test service with chaos injection engine and pre-baked failure scenarios
- SLO engineering — availability (99.5%/30d) and latency (95% under 300ms/7d) with error budget tracking
- Burn-rate alerting — P1 (14x/1h dual window), P2 (1.5x/6h), P3 informational — replacing threshold monitors
- Centralised log management with Datadog Agent, Grok parser, and structured log facets
- Distributed tracing via ddtrace APM — flame graphs, service map, log-to-trace correlation
- Incident response — P1/P2 runbooks, RCA, postmortem documentation

**Stack:** Python · Flask · Datadog · DogStatsD · ddtrace · Datadog Agent · Datadog APM
